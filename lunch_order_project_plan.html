使用角色與權限分工

管理者 Admin

  • 設定每週菜單與餐廳
  • 圖片掃描建立餐點 (AI OCR)
  • 手動進行儲值金加值
  • 檢視所有訂單與帳務明細
  • 截止時間管理與訂單控制

訂購者 User

  • 使用 Google 帳號快速登入
  • 瀏覽當週與預先開放之菜單
  • 開放後至每日 09:40 前可訂餐/退單
  • 即時查看儲值金餘額
  • 瀏覽個人歷史訂單與扣款紀錄

系統排程 Cron

  • 每日 09:40 自動截止當日訂單
  • 餐點確認送達後觸發自動扣款
  • 每週五定時發送下週菜單設定提醒

技術選型與架構規劃

前端 Frontend

Next.js 14 React Tailwind CSS shadcn/ui

使用 App Router、React Server Components (RSC) 以及完全響應式設計，確保極致的使用體驗。

後端 Backend

API Routes Prisma ORM node-cron

基於 Next.js Serverless Function 開發 API；Prisma 做為 ORM 模型，並利用 node-cron 進行定時任務。

資料庫 Database

PostgreSQL Supabase

採用 Supabase 託管的雲端 PostgreSQL 資料庫，支援 Row Level Security (RLS) 保護資料隱私。

第三方服務 Services

Google OAuth Vision API Vercel

串接 Google 登入認證，Vision API 用於菜單圖片文字辨識，全站部署於 Vercel 雲端平台。

預計開發功能模組

Auth Module

認證與授權

串接 Google OAuth 進行單一登入，依據資料庫設定動態判定角色屬性（管理者 / 訂購者），提供安全的 Session 管理。

Restaurant

餐廳與菜單管理

管理者上傳實體菜單圖片，後台整合 Google Vision API OCR 辨識餐廳名稱、分類、品項與價格，由人工確認後一鍵儲存。

Schedule

週菜單排程管理

每週五設定下週五天餐廳；設定完成後立即開放下週整週訂購，支援後台預排及拖曳調整順序。

Order

每日與預先訂購

訂購者可提前瀏覽下週已設定之菜單並預先點餐；系統於每日 09:40 自動截止當日訂單，並發送訂單統計通知。

Billing

儲值金與異動系統

提供個人專屬儲值金帳戶。管理者可手動執行加值，當餐點確認送達後系統自動扣款，並顯示對應明細。

Admin Console

訂單統計與整理

後台彙整當日訂單（依品項進行合計），可一鍵匯出明細方便聯繫餐廳，並可控制「確認送達」以觸發扣款。

資料庫實體關係結構 (主要資料表)

users (使用者)
Primary User Profile Data
id UUID PK 使用者唯一主鍵
email VARCHAR Google 帳號 Email (唯一索引)
name VARCHAR 顯示姓名
avatar_url TEXT Google 帳號頭像圖片位址
role ENUM 角色權限：admin / user
balance INTEGER 儲值金餘額 (預設值為 0，允許負值)
created_at TIMESTAMP 帳號建立時間
restaurants (餐廳)
Restaurant Info & Contact
id UUID PK 餐廳唯一主鍵
name VARCHAR 餐廳名稱
category VARCHAR 餐點類別 (便當、麵食、日式、飲料等)
phone VARCHAR 餐廳聯絡電話
note TEXT 備註事項 (如最低訂購外送金額限制)
is_active BOOLEAN 是否啟用此餐廳 (預設為 true)
menu_items (菜單品項)
Restaurant Menu Details
id UUID PK 品項唯一主鍵
restaurant_id UUID FK 關聯餐廳 (ForeignKey -> restaurants.id)
name VARCHAR 品項名稱
price INTEGER 單價金額
category VARCHAR 品項分類 (如：主食、配菜、飲料)
is_available BOOLEAN 當前是否正常供應中
weekly_schedules (週菜單排程)
Daily Active Restaurant Schedule
id UUID PK 供餐日程唯一主鍵
date DATE 供餐日期 (週一至週五)
restaurant_id UUID FK 當日指定之餐廳 (ForeignKey -> restaurants.id)
order_deadline TIME 訂單截止時間 (預設為 09:40:00)
is_open BOOLEAN 該日期當前是否開放訂購
delivered_at TIMESTAMP 餐點送達點收時間 (以此觸發扣款)

id UUID PK 訂單唯一主鍵

schedule_id UUID FK 對應排程供餐日 (ForeignKey -> weekly_schedules.id)

unit_price INTEGER 下單當下價格快照 (防止後續調整價格影響帳務)

菜單圖片掃描辨識流程 (OCR)

1. 上傳圖片

管理者上傳菜單紙本拍照相片或電子檔

2. OCR 辨識

呼叫 Google Vision API 解析圖中文字區塊

3. AI 結構化

大模型提取菜單名稱、價格、品項分類

4. 管理者確認

後台 UI 顯示結果供管理者編輯修正錯漏

5. 儲存入庫

確認無誤後，正式將餐廳與菜單存入資料表

辨識校對機制：OCR 自動辨識結果必須經過人工最後校對審核，以避免因印刷模糊或字體特殊而導致價格、餐點品項寫入錯誤，確保帳務與出餐之準確性。後台將一併保存菜單原始圖片以供隨時核對。
